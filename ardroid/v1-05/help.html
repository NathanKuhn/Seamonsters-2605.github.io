ARDROID Help

Building the robot

The ARDROID app is designed to connect to an Arduino101 board with the Adafruit Motor Shield V2. The Arduino needs a special sketch installed on it—see the Ardroid GitHub repository for instructions on how to install this sketch. The Adafruit website has instructions on wiring a motor shield—up to 2 stacked motor shields are currently supported to provide extra motors.

Here is a simple example robot on Thingiverse.

Controlling the robot

On the main screen of the app, tap Bot Settings to configure robot settings, like the driving speed and where the motors are plugged in. You must do this before connecting to the robot. The Bot Settings screen has What's this? buttons to explain settings in more detail.

Motor numbers: Each Adafruit v2 motor shield has 4 ports for DC motors, numbered 1 through 4. Stepper motors are numbered 1 and 2, and take up 2 ports each—stepper #1 uses DC ports 1 and 2, and stepper #2 uses DC ports 3 and 4. If you have multiple stacked motor shields, the second shield has DC motor numbers 5 - 8 and stepper motor numbers 3 - 4, and so on.

To connect to the robot, turn on Bluetooth on your phone and press Scan for Bots. Bluetooth devices will show up in a list below. Look for devices with ARDROID in the name. Tap to connect.

Connecting to the robot will open the control screen. The black dot is a joystick, and you can move it to drive the robot (you must have configured the driving motor ports first). You can also tap the Tilt box to allow you to drive the robot by tilting your phone—hold it level to stop moving, and tilt it in any direction to move the joystick in that direction. To stop all motors from moving, tap All Off.

Troubleshooting

General Arduino troubleshooting

  • The Arduino must receive plenty of power for Bluetooth to function. Running motors or leaving the robot on for a while uses power quickly. The Arduino may be low on power, even if the light is on and even if the motors sometimes work. USB by itself is not enough to power motors.
    If you are going through batteries quickly, consider using a power supply. This page has information on what power adapters will work.
  • Make sure that the Arduino has the correct firmware and latest version of the Ardroid sketch. See these instructions for details.
    If you used the robot with a previous version of the app, you may need to update the code on the Arduino.
  • Turn the robot off (unplug the power), wait 5-10 seconds, and turn it on again. This should reset the Bluetooth connection.

The robot doesn't appear in the devices list

  • Make sure that Bluetooth is turned on for your phone, and that your phone supports BLE (Bluetooth Low Energy). Android 4.3 or above is required for BLE.
  • Make sure the robot is on (the green LED on the motor shield should be on).
  • See the "General Arduino troubleshooting" section above.

The app gets stuck while connecting

Connecting should normally take about 5 seconds.

  • Press Cancel and try again.
  • If there are a lot of Bluetooth devices nearby, make sure they all appear in the list before you choose one.
  • Check that you are connecting to the correct bluetooth device—it should have ARDROID in the name.
  • See the "General Arduino troubleshooting" section above.

The Communication light won't turn green

The Communication light is the light to the left of the All Off button—it is red when the app is sending data to the robot, and green when it isn't. If it get's stuck red, there's a problem.

  • Long-press the All Off button for about a second until you see the message below read "Forcing bluetooth communication..."
  • Make sure you connected to the robot and not some other bluetooth device. The device in the list should have "ARDROID" in its name.
  • See the "General Arduino troubleshooting" section above.

The robot doesn't move when you move the joystick

  • Check the Communication light next to the All Off button. This should normally be red when the app is sending data to the robot, and green when it isn't. If it is stuck red and won't turn green, try the steps above under "The Communication light won't turn green".
  • Check that the motor ports are set to the correct values in Bot Settings.
  • Check the connections between the motor shield and the motors.
  • Make sure you connected to the robot and not some other bluetooth device. The device in the list should have "ARDROID" in its name.
  • See the "General Arduino troubleshooting" section above.

Tilt control doesn't work ("no orientation sensor availible")

Some phones don't support tilt control.
